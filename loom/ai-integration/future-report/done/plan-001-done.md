# Done — Future-report v1 — next-work kind + forward/creativity knobs

## Step 1 — Add an explicit `source: 'docset' | 'roadmap' | 'release-notes' | 'forward-signal'` field to the ReportKind interface and tag every existing REPORT_KINDS entry; retire the `docTypes.length === 0` inference (D1).

Added the `ReportSource` discriminator to `packages/core/src/reportKinds.ts`: a new exported type `'docset' | 'roadmap' | 'release-notes' | 'forward-signal'` and a required `source` field on the `ReportKind` interface (documented as the explicit selection-shape signal that retires the brittle `docTypes.length === 0` / `slug === 'release-notes'` inference). Tagged all 10 existing kinds: `project-overview` → `roadmap`; `release-notes` → `release-notes`; `architecture`/`decisions`/`drift-audit`/`security`/`ideas`/`designs`/`plans`/`dones` → `docset`. Exported `ReportSource` from `packages/core/src/index.ts`. No behavior change yet — the prompt still infers shape; step 4 switches it to `kind.source`.

## Step 2 — Create `packages/core/src/forwardSignal.ts`: a pure `(LoomState, ReportFilters) => ForwardSignal` builder that composes existing derivations into the four Tier-1 signal groups with deterministic ranking inputs, and export it + its types from the core index.

Created `packages/core/src/forwardSignal.ts` — the pure Tier-1 builder `buildForwardSignal(state, filters, now=today())`. Composes existing derivations into four grounded signal groups, each item carrying its source doc id + firing detector: **parked-decision** (via `extractParkedSection` — a verbatim `## Open questions`/deferred/parked heading-section lift from idea/design bodies, levels 2–4, no judgement → Tier-1 per D2), **stalled-intent** (graph shape: idea→no design, design→no plan, or a plan that is active/draft but never started), **blocked-work** (blocked plan steps via `isStepBlocked(step, plan, state.index)` + dependency-blocked threads from `buildRoadmap` `blockedOn`), and **drift-debt** (actionable `staleEntries`). Deterministic ranking inputs per item (D4): `leverage` (fan-out proxy — roadmap dependents for threads, dependent-step count for blocked steps, downstream-doc count for parked), `ready` (actionable-now vs itself gated), `ageDays` (created→now). Ranked leverage desc → ready-first → oldest → id; I deliberately did NOT implement leverage×readiness as a literal multiply (it would sink high-leverage blocked items to the bottom — noted in the header comment), using readiness as a secondary sort key instead — within the agreed model. Honours ReportFilters (weave/thread/date, date-window Date-safe via `toCanonical`); emits `isEmpty` for the empty-set stop signal. Exported `buildForwardSignal`, `extractParkedSection`, and the `ForwardSignal`/`ForwardSignalItem`/`ForwardSignalGroup` types from `packages/core/src/index.ts`.

## Step 3 — Add the `next-work` entry to REPORT_KINDS: empty `docTypes`, `source: 'forward-signal'`, cross-weave, with the forward-synthesis promptFraming (ranked next-work list; every proposal cites its signal group; empty-slice stop discipline).

Added the `next-work` entry to `REPORT_KINDS` in `packages/core/src/reportKinds.ts`: `source: 'forward-signal'`, empty `docTypes`, `scopeHint: 'cross-weave'`, title "Next Work". Its `promptFraming` instructs a single ranked next-work list (highest-leverage/unblocked first, respecting the pre-sorted slice), each proposal giving what-to-do / why-now (cite source doc id(s) + firing group) / leverage / first move, with the grounded-observation-free-solution guardrail ("propose a bolder solution, never a new problem") and the empty-set stop discipline (mirrors release-notes' nothing-unreleased guard). Decision refinement vs the design's sketch: the `creativity` clause is NOT a `{creativity}` placeholder token inside the framing — the report prompt appends it generically at run time (step 4), which also lets the same clause compose with `forward` mode on other kinds and avoids a brittle string-replace. Noted in a comment above the entry.
