# Run the loop

Loom's whole workflow is one loop, and every stage is a Markdown document the AI reads and writes — so context never evaporates between sessions:

1. **Chat** — open a chat doc and think out loud with the AI (*AI Reply*).
2. **Idea** — *Promote* the chat to an idea, or create one, then *Refine Idea*.
3. **Design** — *Generate Design* to turn the idea into architecture + decisions.
4. **Plan** — *Generate Plan* to break the design into concrete, reviewable steps.
5. **Implement** — *Do Step* and the AI writes the code, records a done-note, and marks the step ✅.

The **Context** panel shows exactly which documents the AI will receive for the selected node — *before* you launch. What you see is what the AI gets.
